## DevTools CSS Investigation Report

## Selected element

For this investigation I selected the text section inside the weather card showing the current weather.

In DevTools the element is:
div.copy.svelte-1671l1m

This element contains the location name, temperature and weather description.
I chose it because its styling comes from multiple CSS sources such as global styles, component styles and browser default styles.

----

## Investigated CSS properties

**1. color**
   
Computed value
  rgb(255, 236, 252)

Rule shown in DevTools
  :root

Generated CSS location
  app.css:1

Original source
  src/app.css

Explanation:
  The color is defined globally in the :root selector in app.css.
  The element inherits this value from the root styles.

**2. font-family**

Computed value
  Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif

Rule shown in DevTools
  :root

Generated CSS location
  app.css:1

Original source
  src/app.css

Explanation:
  The font is also defined globally in the :root section.
  The element inherits the font from the main application styles.

**3. line-height**

Computed value
  24px

Rule shown in DevTools
  line-height: 1.5

Generated CSS location
  app.css:1

Original source
  src/app.css

Explanation:
  In the CSS source the value is written as 1.5.
  DevTools shows the final computed value in pixels (24px).
  This happens because the browser calculates the real value based on the current font size.

**4. text-align**

Computed value
  left

Rule shown in DevTools
  .copy.svelte-1671l1m

Generated CSS location
  WeatherCard.svelte:62

Original source
  src/components/WeatherCard.svelte

Explanation:
  This rule comes from the component stylesheet inside WeatherCard.svelte.
  Svelte adds a generated class (svelte-1671l1m) to scope styles to this component.

**5. width**

Computed value
  490.888px

Rule shown in DevTools
  No direct rule setting this value.

Generated CSS location
  Not directly defined in one CSS rule.

Original source
  Indirectly comes from layout rules.

Explanation:
  The width shown in DevTools is calculated by the browser based on the layout of the page.
  It depends on the container size and grid layout, so it is not directly written in the source CSS.

----

## Situations where mapping CSS becomes unclear

**1. Generated Svelte selectors**
In DevTools the rule appears as:
  .copy.svelte-1671l1m

The class svelte-1671l1m is automatically generated by Svelte to scope component styles.
In the original source code this class does not appear exactly the same way, which makes the mapping slightly indirect.

**2. Relative values converted to pixels**

For line-height, the source CSS contains:
  line-height: 1.5

But DevTools shows the computed value:
  24px

Because of this conversion, the value seen in DevTools is different from the one written in the source CSS.

**3. Layout-calculated values**

For the property width, DevTools shows:
  490.888px

However this value is not written anywhere in the CSS files.
It is calculated automatically by the browser based on the layout of the page and container size.

----

## Conclusion

This investigation showed how Chrome DevTools can be used to trace computed CSS styles back to their source.
In most cases it is possible to identify the rule and the original file.
However in some situations, such as generated selectors, relative values or layout-based calculations, the mapping between computed styles and original CSS becomes less direct.
